# Prompt Lifecycle

Status: Draft

Version: 0.1

Category: Methodology

---

# Purpose

The Prompt Lifecycle defines how prompts evolve within the AIOS Framework.

Its objective is to ensure that every prompt is validated through real-world usage before becoming part of the official Prompt Library.

AIOS values proven prompts over theoretical collections.

---

# Philosophy

A prompt is considered an engineering asset.

Like software, prompts improve through continuous testing, review, and refinement.

Only prompts that consistently produce reliable results should become part of the framework.

---

# Lifecycle Stages

## Draft

A newly created prompt.

Characteristics:

- Untested
- Experimental
- Subject to frequent changes

---

## Validated

A prompt that has been successfully used in at least one real project.

Characteristics:

- Tested in practice
- Produces reliable results
- Suitable for reuse

---

## Stable

A prompt that has been validated across multiple projects.

Characteristics:

- Well-tested
- Consistent output
- Recommended as the default version

---

## Deprecated

A prompt that has been replaced or is no longer recommended.

Characteristics:

- Kept for historical reference
- Not recommended for new projects

---

# Transition Rules

Draft

↓

Validated

↓

Stable

↓

Deprecated

Progression should always be based on practical experience rather than assumptions.

---

# Evaluation Criteria

A prompt should be evaluated based on:

- Clarity
- Consistency
- Reusability
- Accuracy
- Maintainability
- Practical usefulness

---

# Final Principle

Prompts earn their place in AIOS through successful real-world use.
