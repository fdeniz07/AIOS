# Implementation Strategy

## Purpose

Define the safest and most logical implementation sequence after the repository has been analyzed and its risks have been assessed.

The objective is to organize development into manageable phases that minimize architectural risk and support long-term maintainability.

---

## When to Use

Use this prompt after completing:

1. Repository Analysis
2. Risk Assessment

Run it before any implementation work begins.

---

## Required Context

- Repository Analysis is complete.
- Risk Assessment is complete.
- Repository architecture has been reviewed.
- No implementation has started.
